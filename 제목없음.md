[작업 의뢰서] GEO 및 SRA 메타데이터 전수 수집 파이프라인 구축
1. 프로젝트 목적
NCBI Gene Expression Omnibus(GEO) 및 Sequence Read Archive(SRA) 데이터베이스에 등록된 모든 공공 생물정보학 실험의 메타데이터를 누락 없이(Exhaustive) 수집, 파싱하여 통합 데이터베이스를 구축한다.

2. 필수 수집 대상 및 항목 (Scope of Work)
단순한 텍스트 긁어오기가 아닌, 각 Accession Number에 종속된 모든 하위 메타데이터 속성을 정형화하여 추출해야 합니다.

A. GEO 메타데이터 수집 항목

GSE (Series - 연구/프로젝트 단위):

연구 제목(Title), 요약(Summary), 전체 실험 디자인(Overall Design).

연구 책임자 및 기여자 정보(Contributor, Contact).

관련 출판물 정보(PubMed ID / Citation).

등록일(Submission date) 및 최종 수정일(Update date).

GSM (Sample - 개별 샘플 단위):

샘플 특성(Characteristics): 조직 종류(Tissue), 세포주(Cell line), 발달 단계(Developmental stage), 질환 상태(Disease state), 연령, 성별 등 임상/생물학적 정보 전수.

처리 조건(Treatment protocol): 약물 처리, 유전자 조작 등.

추출 프로토콜(Extraction protocol) 및 라벨링 정보.

GPL (Platform - 실험 기기 단위):

제조사(Manufacturer), 장비 명칭, 어레이 기술 제원.

생물종(Organism).

B. SRA 메타데이터 수집 항목 (SRA XML 기반)

SRP (Study): GEO의 GSE와 매핑되는 전체 연구의 백그라운드 및 목적.

SRS (Sample) 및 BioSample 연동: * 분류학적 정보(Taxonomy ID), 식별자, 구체적인 생물학적 출처 메타데이터.

SRX (Experiment - 라이브러리 및 기기 세팅):

시퀀싱 전략(Strategy): WGS, RNA-Seq, ChIP-Seq, scRNA-Seq 등.

소스(Source): Transcriptomic, Genomic 등.

선택 방식(Selection): polyA, Random 등.

레이아웃(Layout): Paired-end 또는 Single-end 여부 및 Insert size.

장비 모델(Instrument model): 예) Illumina NovaSeq 6000.

SRR (Run - 실제 데이터 파일 단위):

총 리드 수(Total Spots/Reads), 총 염기쌍 수(Total Bases).

파일 크기(Size), 평균 리드 길이(Average Length).

원시 데이터 다운로드 경로(FTP, AWS, GCP 링크).

3. 데이터 수집 방식 및 기술 요구사항
공식 API 및 Bulk 데이터 활용: 웹 스크래핑 방식은 지양하며, NCBI E-utilities (esearch, efetch, elink) API를 기본으로 사용한다. 대규모 데이터의 경우 NCBI FTP 서버의 SRA 메타데이터 전체 덤프(XML)를 활용하여 초기 구축을 진행한다.

ID 매핑 (Cross-referencing): GEO의 GSE/GSM과 SRA의 SRP/SRR, 그리고 BioProject, BioSample, PubMed ID 간의 연결 고리(Linkage)를 완벽하게 유지하여 관계형 데이터 구조로 추출해야 한다.

오류 제어 및 지속성: API 호출 제한(Rate Limit)을 준수하고, 연결 끊김 시 진행 상태를 기억하여 재개(Resume)할 수 있는 로직을 포함한다.

4. 데이터베이스 저장 규격
수집된 메타데이터는 비정형 텍스트를 최소화하고 JSON 형태로 파싱하거나 관계형 데이터베이스(예: MySQL, PostgreSQL)의 테이블 스키마에 맞게 정규화(Normalization)하여 적재한다.

가변적인 샘플 특성(GSM Characteristics) 데이터는 Key-Value 형태의 JSON 데이터 타입으로 유연하게 저장할 수 있도록 설계한다.

5. 산출물 (Deliverables)
메타데이터 추출 자동화 파이프라인 (소스 코드).

구축된 데이터베이스의 테이블 스키마 (ERD).

누락된 항목이나 파싱 에러를 기록한 로그 파일 (QC Report).
